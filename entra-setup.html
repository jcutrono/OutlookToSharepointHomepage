Entra ID Setup

Configure Microsoft Entra ID roles to control who can use FileMy.Email and who can manage your organization's seats and subscription.

Finding your Microsoft 365 Tenant ID

When you subscribe on the pricing page, Stripe checkout asks for your Microsoft 365 Tenant ID. This is what links your payment to your organization, so it has to be entered exactly.

  1. Go to the Microsoft Entra admin center and sign in with a work account.
  2. On the Overview page, find Tenant ID under Basic information.

    It's a 36-character value that looks like 77a5b88f-a3d1-4957-9b08-73d172054a46.

  3. Click the copy icon next to it, then paste it into the Microsoft 365 Tenant ID field at checkout.

    Copy and paste rather than typing it out — a single wrong character means your payment can't be matched to your organization and access won't turn on.

Not an admin? Ask whoever manages your Microsoft 365 tenant for the Tenant ID, or have them complete the purchase.

How access works

FileMy.Email uses two app roles in Microsoft Entra ID to control access:

FileMyEmail.User

Required for all users

Allows the user to save emails and attachments to SharePoint. Each user with this role consumes one seat in your subscription.

FileMyEmail.Admin

Optional — for tenant administrators

Allows the user to view subscription details, manage seated users, and remove users who have left the organization. Admins do not consume a seat unless they also have the User role.

Note: Microsoft 365 Global Administrators and SharePoint Administrators automatically have admin access without needing the FileMyEmail.Admin role assigned.

Step 1: Assign roles to users

You assign roles through the Enterprise Applications section of the Microsoft Entra admin center.

  1. Go to the Microsoft Entra admin center.

    Sign in with a Global Administrator or Privileged Role Administrator account.

  2. Navigate to Identity → Applications → Enterprise applications.
  3. Search for "OutlookToSharePoint" and click on the application.
  4. In the left sidebar, click Users and groups.
  5. Click Add user/group.
  6. Under Users, select the user or group you want to assign.

    Assigning a security group gives all members of that group the role automatically. This is the easiest way to manage access at scale.

  7. Under Select a role, choose FileMyEmail User or FileMyEmail Admin.
  8. Click Assign.

Repeat for each user or group. A user can have both roles — they will have admin access and consume a seat.

Step 2: Grant API permissions (one-time)

The add-in uses several delegated Microsoft Graph permissions. Most are requested automatically when a user first signs in; the ones below benefit from a one-time admin consent so users don't see extra prompts.

  1. In the Entra admin center, go to Identity → Applications → App registrations.
  2. Search for "OutlookToSharePoint" and click on the application.
  3. In the left sidebar, click API permissions.
  4. Verify that the following delegated Microsoft Graph permissions appear in the list:
    • User.ReadBasic.All — so admins can see display names for seat management.
    • Mail.Read, Mail.ReadWrite, Sites.Read.All, Files.ReadWrite.All, User.Read — so the add-in can read email and save to SharePoint on the user's behalf.
    • offline_access — required for Auto-Archive. Lets the service keep filing matching mail while a user is out of office.
    • MailboxSettings.Read — required for Auto-Archive. Lets the service check the user's Out-of-Office state.

    If any are missing, click "Add a permission" → Microsoft Graph → Delegated permissions → search for the name → Add. All are delegated (per-user), not application-level — the service always acts as the signed-in user, not tenant-wide.

  5. Click Grant admin consent for [your organization].

    This button appears at the top of the permissions list. You need Global Administrator privileges for this step. Skipping it is fine — individual users will be prompted to consent the first time they enable Auto-Archive.

Common scenarios

Give everyone in my company access: Create a security group containing all employees (or use an existing "All Users" group). Assign the FileMyEmail User role to that group.

Designate an admin: Assign the FileMyEmail Admin role to the person who should manage seats. They will see an "Admin" tab in the add-in where they can view subscription status, see who is using each seat, and remove users.

Remove a departed employee: An admin can remove a user's seat directly from the Admin tab in the add-in, with the option to purge their saved data. You can also remove their role assignment from Entra to revoke access.

Add more seats: Admins can click "Manage Subscription" in the Admin tab to go to the Stripe billing portal, where they can increase the seat quantity on your plan.

You're all set

Once roles are assigned, users will see the FileMy.Email button in their Outlook ribbon. Admins will see an additional "Admin" tab for managing seats and subscriptions.

The add-in may take a few minutes to recognize newly assigned roles. If a user doesn't have access immediately, ask them to close and reopen Outlook.
