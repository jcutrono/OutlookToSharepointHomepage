The paper trail every property manager wishes they had

Properties accumulate email like nothing else. Every unit produces tenant complaints, vendor invoices, inspection photos, lease addenda, insurance certificates, board approvals, and contractor quotes. The official rule is "every email gets filed to the property folder." The reality is that nobody has time, so:

  • Tenant complaints live in someone's inbox until they escalate
  • Vendor invoices get forwarded to accounting with no attachment to the property record
  • Inspection reports stay on a property manager's phone in their email
  • Lease addenda are signed and emailed back, but never make it to the property file
  • When ownership transfers or a property is sold, the file is wildly incomplete

This add-in routes email and attachments to the right property folder in SharePoint without changing how anyone works in Outlook. It learns which folder maps to which tenant or vendor. Save on Send catches the outgoing replies. Mobile staff can file from the field.

Common scenarios this solves

Tenant maintenance request

Tenant emails maintenance@. Add-in suggests the property's tenant folder. One click — request and any photos file to SharePoint. Maintenance team replies and Save on Send files the outgoing message in the same folder. Full thread is in the property file.

Vendor invoice intake

Plumber emails an invoice PDF. Add-in suggests the property's vendor folder. Saves with name "{YYYY-MM-DD} - {Vendor} - {Subject}.pdf" and metadata columns set so accounting can pull a property's invoices in seconds.

Inspection from the field

Property manager finishes a unit inspection on their phone, emails the report to themselves with photos attached, opens the add-in in Outlook mobile, and files everything to the unit's folder before leaving the parking lot.

Lease documents

New lease arrives signed via email. One click files the email and the signed PDF to the unit folder. Save on Send catches the welcome email going back to the tenant.

Property turnover or sale

When a property changes hands, the entire correspondence record exists in the property's SharePoint folder — searchable, exportable, and complete. No reconstructing from scattered inboxes.

Frequently asked questions

Can I file by property address?

Yes. Set SharePoint folders up by property — Property / Unit / Tenant is a common structure. After saving once, the add-in suggests the same folder for future mail with that contact.

How does this compare to AppFolio or Buildium?

AppFolio and Buildium handle leasing, accounting, and maintenance tickets — they're not built for filing everyday email. This add-in fills the email-filing gap by routing mail to your SharePoint property folder, alongside whatever PMS you already use.

Can field staff file from a phone?

Yes. The add-in works in Outlook on the web and the Outlook mobile app — agents and inspectors file from the field.

What about leasing@ or maintenance@ shared inboxes?

The add-in works inside any mailbox the user has access to in Outlook. Each user occupies one seat regardless of mailboxes.

Can I save vendor invoices with metadata for accounting?

Yes. Map SharePoint columns to email metadata once. Every saved invoice has Vendor, Property, and Invoice Date populated for filtering and reports.

Does it work across multiple property entities?

Yes. The add-in handles multiple SharePoint sites, so a management company with several ownership entities can file each property's mail to the entity's own SharePoint without switching accounts.
