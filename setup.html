Installation

Follow the steps below to deploy the add-in, then head to Entra ID Setup to configure user access and admin roles.

Choose your installation method

How you install depends on your role. Pick the option that applies to you:

Option A: IT Admin — Deploy to your entire organization

If you are a Microsoft 365 admin and want to roll out the add-in to multiple users at once, use centralized deployment.

Copy the manifest URL below — you'll paste it into the admin center.

https://app.filemy.email/manifest.xml
  1. Go to the Microsoft 365 admin center.

    Sign in with your admin account.

  2. Navigate to Settings → Integrated apps.

    If you don't see this option, go to Settings → Add-ins instead.

  3. Click Upload custom apps.
  4. Select "Provide link to manifest file" and paste the manifest URL from above.
  5. Choose which users or groups should have access to the add-in.

    Your subscription includes seats for the users in your plan.

  6. Click Deploy and wait for deployment to complete.

    It may take up to 24 hours for the add-in to appear in users' Outlook.

Option B: Individual User — Install for yourself

If you are installing the add-in just for your own Outlook account, follow these steps.

First, download the manifest file:

manifest.xml Download
  1. Open Outlook Add-ins (this link opens the add-ins panel directly).
  2. Click My add-ins in the left sidebar.
  3. Under "Custom add-ins", click Add a custom add-in → Add from file.
  4. Select the manifest.xml file you downloaded above and click Install.

Next: Configure Entra ID

After installing the add-in, you need to assign user roles in Microsoft Entra ID so your team can access the service. This also lets you designate admins who can manage seats directly from the add-in.

Continue to Entra ID Setup →

Optional: Enable Auto-Archive

Once the add-in is installed and a user has set up at least one mapping rule (sender → SharePoint folder), they can enable hands-off filing from the add-in's Settings tab.

  1. Open the add-in inside Outlook and click Settings.
  2. Scroll to the Auto-Archive section.
  3. Pick one or both:
    • Run automatically while I'm Out of Office — uses your Outlook Automatic Replies setting as the trigger. Best for vacation/PTO.
    • Always run automatically — files matching mail as soon as it arrives, regardless of OOF.
  4. Click Apply Auto-Archive. The first time, Microsoft 365 will prompt for two additional permissions (offline_access and MailboxSettings.Read) so the service can keep filing while the user is offline and check their OOF state.

Only emails matching an existing mapping rule are auto-archived — nothing is filed by surprise. Either toggle can be turned off at any time; doing so tears down the background subscription immediately.
